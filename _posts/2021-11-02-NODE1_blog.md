<h3 id="abstract">Abstract</h3>
<p>Today we’ll talk about the popular <a href="https://arxiv.org/abs/1806.07366">Neural ODE</a> paper which won best paper award in Neurips 2018.  This paper develops a foundational framework for infinitely many layered deep neural networks. The framework allows us to take advantage of the extensive research on ODE Solvers.</p>
<hr>
<h3 id="background--ordinary-differential-equations">Background : Ordinary Differential Equations</h3>

<h3 id="infinite-layers-ode-solver-as-foward-pass">Infinite layers: ODE Solver as foward pass</h3>
<p>Did you ever think that a DNN can have infinite layers? If we try to implement it naively by having different parameters in each layer, then there are two prominent issues we’ll be facing,<br>
i)Overfitting<br>
ii)Memory issues. This is because each layer with learnable parameters in DNN needs to store its input until the backward pass. So, it’s not practically feasible in traditional setting.</p>
<p>A natural question arises that “why can’t we have infinite updates in ODE solver instead of <span class="katex--inline"><span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><mrow><mi>l</mi></mrow><annotation encoding="application/x-tex">l</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height: 0.69444em; vertical-align: 0em;"></span><span class="mord mathnormal" style="margin-right: 0.01968em;">l</span></span></span></span></span> updates?”, where <span class="katex--inline"><span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><mrow><mi>l</mi></mrow><annotation encoding="application/x-tex">l</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height: 0.69444em; vertical-align: 0em;"></span><span class="mord mathnormal" style="margin-right: 0.01968em;">l</span></span></span></span></span> denotes number of euler steps or layers. We just need to use state of the art adaptive ODE Solvers which is memory efficient (more discussed later).</p>

<h3 id="results">Results</h3>
<p><strong>Experiment 1 : Supervised Learning</strong><br>
The paper performs a supervised learning on MNIST data. It uses resnet and multi-layered perceptron as baselines. The result is shown below,</p>
<p><img src="https://bella.cc/ox-hugo/screenshot_2019-09-11_15-40-28.png" alt=""></p>
<p>RK-Net uses Runga kutta for forward pass and automatic differentiation for backward pass. We can see that ODENet has fewer parameters with similar accuracy and constant memory.</p>
<p><strong>Experiment 2 : Normalizing flows</strong><br>
The paper proposes a continuous version of normalizing flows. Traditionally normalizing flows is enabled using change of variable formula. This paper proposes instantaneous change of variables formula using ODE dynamics.</p>
<p><img src="https://i.ytimg.com/vi/3-Jmfj4ToX8/maxresdefault.jpg" alt=""></p>
<p>The results are shown belo,</p>
<p><img src="https://bella.cc/ox-hugo/screenshot_2019-09-11_16-14-38.png" alt=""></p>
<p><strong>Time Series Latent ODE</strong><br>
Traditional Rnns can only utilize regular time interval signals in its vanilla form. Neural ODE allows us to sample from a continuous dynamic system,</p>
<p><img src="https://miro.medium.com/max/1113/1*LRtGTST73BgRE33BgXS2AA.png" alt=""></p>
<p>The result for both the RNN and neural ODE for continuous time series for a spiral synthetic data is shown below. It can be seen that RNNs learn very stiff dynamics and have exploding gradients while ODEs are guaranteed to be smooth.</p>

It’s impact on learning dynamics for physical and biophysical system would be immense in future.</p>
